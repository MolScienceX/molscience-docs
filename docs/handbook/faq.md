# 常见问题

本页整理新成员加入 MolScience 时经常遇到的问题。

## 我需要编程经验吗？

编程经验有帮助，但不是所有任务都需要。新成员可以先从文档、文献整理、数据检查、翻译或 GitHub 基础协作开始。

## 我应该先做什么任务？

优先选择范围小、结果明确的任务。例如修正文档中的错别字、补充一段说明、整理一个数据来源、检查一个字段的单位，或者复现一个已有示例。

## 不确定内容是否正确怎么办？

不要直接把不确定内容写成结论。可以在 issue 中提问，或在文档中用“待确认”标注，并附上当前依据和需要确认的问题。

## 中英文页面需要完全一致吗？

核心信息应保持一致，包括标题、流程、数据要求和链接。表达方式可以根据语言习惯调整，但不要让两个版本出现相互矛盾的说明。

## 数据来源应该记录到什么程度？

至少记录来源名称、链接或文献信息、获取时间、字段含义、单位和已知限制。无法确认的信息要显式标注，不要留成空白含义不明的值。
