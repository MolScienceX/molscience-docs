# 数据库指南

本页介绍 MolScience 数据库相关工作的基本流程和记录规范。

## 数据库用途

数据库用于组织分子相关信息、实验记录、计算性质和可用于模型训练的数据。它的价值不仅在于保存数据，还在于让数据来源、字段含义和质量状态可追踪。

## 基本原则

- 字段名称保持一致，避免同义字段重复出现。
- 每条数据都应记录来源，能链接到文献、网页、实验记录或计算脚本。
- 数值必须写明单位，单位转换要保留转换规则。
- 缺失、不确定或不适用的信息要显式标注。
- 修改数据时记录原因，避免无法追溯的覆盖。

## 常见字段

- 分子标识：名称、SMILES、InChI、CAS 号或内部 ID。
- 来源信息：文献、数据库、实验记录、提交人和获取时间。
- 性质字段：性质名称、数值、单位、条件和测量方法。
- 质量字段：是否已检查、可信度、异常说明和待确认问题。

## 数据检查清单

1. 分子标识是否唯一且可复核。
2. 单位是否明确，是否需要统一换算。
3. 数据来源是否足够具体。
4. 是否存在重复、冲突或异常值。
5. 是否记录了不确定性和限制条件。
