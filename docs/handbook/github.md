# GitHub 协作

MolScience 使用 GitHub 管理文档、数据说明、代码和协作讨论。本页介绍新成员需要掌握的最小工作流。

## 基本流程

1. 阅读相关文档或 issue，确认任务范围。
2. 从主分支创建新分支。
3. 在本地完成修改，并用清晰的提交信息记录变化。
4. 推送分支并创建 pull request。
5. 根据 review 意见修改，直到合并。

## Issue 使用建议

- 用 issue 记录待办事项、问题、讨论和可复现的错误。
- 标题要简短明确，例如“补充数据库字段说明”。
- 正文说明背景、预期结果、当前问题和相关链接。
- 不确定是否值得做时，也可以先开 issue 讨论。

## Pull request 要求

- 一个 pull request 只解决一个主题。
- 描述中写清楚修改了什么、为什么修改、如何检查。
- 涉及数据或结论时，附上来源或复现方式。
- 合并前确认链接、标题和中英文页面是否同步。

## 提交信息

建议使用简短的动词开头，例如：

- `docs: update onboarding checklist`
- `data: clarify molecule field units`
- `fix: correct broken handbook link`
