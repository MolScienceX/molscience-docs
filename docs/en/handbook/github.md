# GitHub Guide

This page explains how MolScience uses GitHub for collaboration.

## Basic workflow

1. Read the relevant documentation or issue and confirm the task scope.
2. Create a new branch from the main branch.
3. Make local changes and record them with clear commit messages.
4. Push the branch and open a pull request.
5. Respond to review comments until the pull request is ready to merge.

## Issues

- Use issues to track tasks, questions, discussions, and reproducible bugs.
- Keep titles short and specific, such as "Add database field descriptions".
- Include background, expected outcome, current problem, and related links.
- If you are unsure whether a change is worth doing, open an issue first.

## Pull requests

- Keep each pull request focused on one topic.
- Explain what changed, why it changed, and how it was checked.
- Include sources or reproduction steps when data or conclusions are involved.
- Before merging, check links, headings, and bilingual page consistency.

## Commit messages

Use short action-oriented messages, for example:

- `docs: update onboarding checklist`
- `data: clarify molecule field units`
- `fix: correct broken handbook link`
