# FAQ

Frequently asked questions for new MolScience members.

## Do I need programming experience?

Programming experience is useful, but not required for every task. New members can start from documentation, literature review, data organization, or basic GitHub collaboration.

## What should I work on first?

Choose a small task with a clear result. Good first tasks include fixing documentation typos, adding one explanation, organizing one data source, checking field units, or reproducing an existing example.

## What should I do when I am unsure?

Do not write uncertain content as a final conclusion. Open an issue, or mark the content as "to be confirmed" and include the current evidence and the question that needs confirmation.

## Do Chinese and English pages need to match exactly?

The core information should stay consistent, including headings, workflows, data requirements, and links. Wording can follow each language's style, but the two versions should not contradict each other.

## How much detail should a data source record include?

At minimum, record the source name, link or citation, access date, field meaning, units, and known limitations. Mark unknown information explicitly instead of leaving ambiguous blank values.
